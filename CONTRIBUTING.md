# Contributing Guide

All contributors — human or AI-assisted — must follow these guidelines.

## Required Reading

Before contributing, read:

- `docs/system-overview.md`
- `docs/architecture-rules.md`
- `docs/coding-rules.md`
- `docs/development-workflow.md`
- `PROJECT_MAP.md`

## Workflow

All work must follow the spec-driven workflow:

1. Read system documentation
2. Identify the relevant module spec
3. Review the execution checklist
4. Implement the smallest vertical slice
5. Validate the implementation
6. Update documentation if necessary

## Core Rules

- Respect Clean Architecture
- Respect Clean Code
- Do not implement features not described in a spec
- Do not break documented layer boundaries
- Review all AI-generated code before accepting it
